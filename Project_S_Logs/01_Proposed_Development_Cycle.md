# Proposed Development Cycle

Proposed by Saad Shafique for Product S. This document outlines the high-level process and stage-by-stage strategy for building the platform.

---

## Stage 1: Development and Research on Open-Source That Best Fit Our Use Case

* Need to be Web accessible
* Must be Popular and/or be widely appreciated
* MIT License is mandatory for native integrations
* Each Integration must provide a complete Solution for that service

## Stage 2: Initial Setup — User Side

Initial Setup should be simple however it should be robust enough to set a standard for industry security.

* We ask User to randomize a key by their mouse movements in the browser one time for initial setup so that every user has a Unique Key
* We ask for Master Login and Password on setup which can not be changed and must be saved encrypted using the unique key generated by the users mouse movements which will be tracked for a total of one time
* Once the Master User is Created he can create as many User Accounts as needed and assign them privileges as they please
* If a big team is Using the Service they can assign pre existing roles to different Users as needed without configuring everything again for a different User

## Stage 3: Designing the Dashboard

* It should be simple but it should also provide the complete control of the Server
* Mainly it should focus on no command line but should also have a module for command line
* Widget Development for Server Specs (RAM, Usage, CPU, GPU, Storage)
* Logo and Brand Identity

## Stage 4: Phase 1 — The Basics (NAS, Drive, CodeSpace, Matrix, Password Storage, Nginx, Docker in Docker, Git) First Iteration

* **NAS/Media:** Proposed Solution is JellyFin because it can Store Videos, Audio and Images — https://github.com/jellyfin/jellyfin.git
* **Google Suite Replacement:** Proposed option is NextCloud because of its versatility — https://github.com/nextcloud/server.git
* **CodeSpace:** Theia is suggested which is a VS code fork with a web-ui with complete featureset and MIT License https://github.com/eclipse-          theia/theia.git
* **Deployment and Testing:** With the CodeSpace we also need to provide tools necessary for simulation for different devices
    * Selenium — https://github.com/SeleniumHQ/selenium.git
    * For Mobile Device Testing Expo with command line control — https://github.com/expo/expo.git
* **Conversation:** Integrate Matrix for an Open Source communication server — https://github.com/matrix-org/synapse.git
* **Reverse Proxy:** Nginx for mapping ports to different services — https://github.com/nginx/nginx.git — GUI suggestion: https://github.com/0xJacky/nginx-ui.git
* **Docker in Docker:** To create own applications for personal uses
* **Git:** For version Control — https://github.com/git/git.git
* **Mobile:** React Native Application that connects to server

## Stage 5: Phase 2 — The Nice to Haves (ERP, n8n, Kiwix) Second Iteration (business variant)

* **ERP:** Can use ERPNext, a tried and true method for managing a business — optional and charged a premium if used on our servers, however for self hosting it will be one click away
* **Knowledge Base:** Kiwix will be used for storing the internet knowledge as we know it
* **Mail Server:** ERP has its own, but another valid option is ERPNext own server and NextCloud for creating your own mail with self hosting
* **Automation:** User can download n8n in their own environment with the click of a button which can git pull and compose it so that User can set up their own environment

## Stage 6: Supply Chain Exploration (Open-Source Smart Home)

**Tagline:** A smart home that does not spy on you

* Explore Chinese Markets for smart-home devices that can create smart Arduino based devices
* Propose to sell them on an online marketplace
* Inventory Management with the software
