# Project S — Design Extension (v6.2)

This document serves as an extension to the Dashboard Technical Report (06), detailing the transition from a standard web dashboard to a functional, multi-app "Operating System" interface.

---

## 1. The "HomeForge OS" Vision

The design has evolved from a collection of service links into a unified desktop environment. The core philosophy is **"One Interface, Many Apps,"** where third-party services like Nextcloud and Matrix feel like native system modules rather than external sites.

### Key UX Shift:
- **Old Way:** Navigation replaced the entire page content. Services were isolated islands.
- **New Way:** Services open in individual windows that can be minimized or closed, allowing the user to maintain state across multiple applications.

---

## 2. Component: Windowing System (`WindowWrapper`)

Every integrated application is now wrapped in a high-fidelity windowing component.

### 2.1 Window Controls
We moved away from OS-specific styles (like macOS traffic lights) to a custom minimalist set:
- **Minimize (—):** Hides the window and grays out the icon in the dock.
- **Close (X):** Destroys the application state and removes it from the dock.
- **Maximize:** Currently reserved for future multi-tasking layouts.

### 2.2 Fluid Transitions
Transitions use a custom **`cubic-bezier(0.4, 0, 0.2, 1)`** curve.
- **Entrance:** Apps "float up" from the dock with a slight scale and opacity fade.
- **Exit:** Apps descend and fade, giving a sense of physical space.

---

## 3. Component: The Mac-Style Dock (Sidebar)

The sidebar has been refactored to act as a system dock.

### 3.1 Core Navigation
Only the "System Essentials" are permanently pinned:
- **Home:** The primary application launcher.
- **Metrics:** Real-time system health and resource monitoring.
- **Terminal:** Direct SSH/Docker command access.
- **Settings:** Theme and preference management.

### 3.2 Dynamic Recent Apps
When a user launches an app from the Home grid, it is dynamically added to the **Dock Section** of the sidebar.
- **Active Indicator:** A subtle white dot appears under running apps.
- **Minimized State:** Minimized apps stay in the dock but are desaturated to indicate they are "sleeping" in the background.

---

## 4. Infrastructure: The Proxy Gateway (Nginx)

To enable the windowing system, we implemented a sophisticated proxy layer.

### 4.1 Header Stripping
Many services (Nextcloud, Jellyfin) use `X-Frame-Options: SAMEORIGIN` which blocks them from loading in iframes.
- **The Fix:** An Nginx reverse proxy intercepts these ports and **strips the security headers** before they reach the browser.
- **Result:** Complete visual integration without compromising local service security.

### 4.2 Standardized Port (3069)
The entire ecosystem has been migrated to port **`3069`**.
- **Dashboard:** `localhost:3069/`
- **Services:** Proxiable via `localhost:3069/path/` or intercepted via their native ports through the proxy.

---

## 5. Ultimate Metrics Architecture (Container-Native Monitoring)

To bring the System Monitor to life, we implemented a container-first data pipeline.

### 5.1 The Docker Socket Bridge
- **Mount:** `/var/run/docker.sock` is mounted directly into the Dashboard container.
- **Implementation:** The Dashboard runs as `root` to allow binary execution of the `docker stats` command from within the Next.js API.

### 5.2 Real-Time Data Points
- **Aggregate Load:** Calculates total CPU/RAM used by the Project S stack.
- **Storage Distribution (NEW):** A Pie Chart analyzing the physical disk usage of `./data/` folders (Nextcloud, Jellyfin, etc.).
- **Resource Consumers (NEW):** A Bar Chart ranking containers by their current power consumption.
- **Traffic Pulse (NEW):** A dual-axis Area Chart showing real-time Network In (Download) and Out (Upload) for all services.
- **Vibe Jitter:** To ensure the dashboard always feels "alive," a simulated background load (3-5%) is added when the system is perfectly idle.
